# TrainHub 冷启动 Playbook

> "Do things that don't scale." -- Paul Graham
>
> 这不是一份营销方案，这是一份作战手册。每一步都是具体动作，不是战略空话。

**当前状态**：产品已上线（Mock 模式），Revenue $0，Users 0
**目标**：30 天内完成供给侧 30 位培训师 + 需求侧 5 家种子企业，产生第一笔撮合收入
**市场**：中国企业培训市场，目标用户为企业 HR/培训负责人 + 自由培训师

---

## 7-Day Completion Sprint

> 目标：7 天内把项目从“可演示”推进到“可持续运营”，优先完成工程基线、询盘闭环、运营留痕、索引执行。

### Day 1：清理工程阻塞

- 动作：修复 `lint` 错误与 warning，确保 `npm run lint` 通过
- 命令：
  - `npm run lint`
  - `npm run build`
- 验收标准：两条命令都通过，且无 ESLint error
- 回滚方式：`git restore <modified-files>` 或回退到前一提交

### Day 2：建立发布闸门

- 动作：新增 `qa:release`（统一执行 `lint + build`），并接入 CI
- 命令：
  - `npm run qa:release`
  - 检查 workflow 是否使用 `qa:release`
- 验收标准：本地 `qa:release` 通过，CI 配置命中
- 回滚方式：移除新增脚本并恢复 workflow 原构建步骤

### Day 3：询盘提交模式显式化

- 动作：询盘返回 `api / external-form / local-fallback`，前端按模式提示
- 命令：
  - `npm run lint`
  - `npm run build`
- 验收标准：编译通过，且本地兜底模式有明确文案
- 回滚方式：恢复 `src/services/api.ts` 和 `InquiryModal.tsx` 到前一版本

### Day 4：部署环境变量接入

- 动作：在 GitHub Actions 发布步骤注入 `VITE_FORM_ENDPOINT`
- 命令：
  - 检查 `.github/workflows/deploy-pages.yml` 中 `env.VITE_FORM_ENDPOINT`
  - 手动触发一次 workflow（如有权限）
- 验收标准：工作流读取到 Secret 且构建不报缺失
- 回滚方式：移除该 env 注入，恢复为纯本地兜底模式

### Day 5：运营周报基线

- 动作：建立 `docs/operations/weekly/` 与当周周报 `2026-W10.md`
- 命令：
  - `rg "docs/operations/weekly|2026-W10" docs/operations -n`
- 验收标准：周报目录、README、首周报三项齐全并可检索
- 回滚方式：删除新增周报文件并恢复指标文档索引

### Day 6：索引执行留痕

- 动作：创建索引提交执行日志，记录 GSC/Bing 每个动作的证据字段
- 命令：
  - `rg "Google Search Console|Bing Webmaster|执行人|截图|结果" docs/research/indexing-submission-log-2026-03-03.md -n`
- 验收标准：日志包含任务清单、时间戳、截图路径、结果链接字段
- 回滚方式：删除日志文件并移除审计文档中的引用

### Day 7：周复盘与下周计划

- 动作：复盘 7 天执行结果，更新下周优先级与量化目标
- 命令：
  - `npm run qa:release`
  - 更新并检查 `docs/operations/weekly/2026-W10.md`
- 验收标准：发布闸门通过，周报含“已完成/未完成/下周动作”
- 回滚方式：保留工程改动，单独回退周报中的结论性内容

## 第一部分：供给侧 -- 培训师获取

### 第 1-7 天：手动招募前 10 位培训师

**核心原则**：不要发广告，不要群发消息。一个一个找，一个一个聊。像 Airbnb 创始人亲自给房东拍照一样，你要亲自帮培训师完善主页。

#### 渠道 1：脉脉（优先级最高）

**为什么脉脉最优先**：脉脉是中国最大的职场社交平台，培训师集中度高，可以按职业/技能搜索，且用户习惯接受陌生人的职业对话。

**搜索关键词**（按优先级排序）：
1. "企业培训师" -- 最精准的身份标签
2. "领导力培训" / "销售培训" / "数字化转型培训" -- 对应 TrainHub 六大品类
3. "企业内训" + "自由职业"
4. "TTT培训" -- 培训培训师的人，通常自己也是资深培训师
5. "组织发展顾问" / "人才发展顾问"

**筛选标准**（必须同时满足至少 3 条）：
- [ ] 从业经验 8 年以上
- [ ] 有明确的专业领域（不是什么都教的万金油）
- [ ] 个人简介中提到具体服务过的企业或行业
- [ ] 有内容输出习惯（发过文章/动态）
- [ ] 目前是自由职业或独立顾问状态（比在职培训师更需要获客渠道）

**联系话术**：

```
Hi [姓名]，

看到您在[具体领域]的分享，特别是[引用对方某条具体内容]，很有共鸣。

我在做一个培训师技能市场 TrainHub（trainhub.withpace.com），帮企业找到靠谱的培训师，
也帮培训师获得更多企业客户。

目前在邀请行业里有口碑的培训师首批入驻，想了解您是否有兴趣？
入驻完全免费，我来帮您搭建在线课程主页。

方便加个微信聊聊吗？
```

**关键点**：
- 必须引用对方的具体内容，证明你是认真看过的，不是群发
- 不要一上来就推销平台，先建立信任
- "我来帮您搭建" 这句话是关键 -- Do things that don't scale

**每日行动量**：
- 发送 20 条个性化消息
- 预期响应率 15-20%（3-4 人回复）
- 预期转化率 50%（1-2 人愿意入驻）
- 7 天目标：联系 140 人，获得 10 位培训师

#### 渠道 2：LinkedIn

**搜索关键词**：
1. "Corporate Trainer" + "China"
2. "企业培训" + "顾问"
3. "Leadership Development" + "Shanghai/Beijing/Shenzhen"

**筛选标准**：
- 中英双语资料（说明国际化背景，溢价能力强）
- 有 500+ 连接（说明在行业内有人脉）
- 近 3 个月有发布内容

**话术（LinkedIn 版本）**：

```
[姓名]您好，

拜读了您关于[具体话题]的文章，对[某个观点]特别认同。

我在做 TrainHub，一个连接优质培训师和企业客户的平台。
目前邀请首批培训师入驻，希望您有兴趣了解？

平台链接：https://withpace.github.io/trainhub/

期待交流！
```

**每日行动量**：发送 10 条消息（LinkedIn 有发送限制）

#### 渠道 3：知乎

**搜索方式**：
1. 搜索话题 "企业培训"、"管理培训"、"领导力" 下的高赞回答者
2. 关注 "培训师" 话题，找到活跃回答者
3. 搜索问题 "如何成为企业培训师"、"自由培训师如何获客"

**筛选标准**：
- 回答数 > 20 且涉及培训领域
- 有个人签名或简介表明培训师身份
- 内容质量高，不是水军

**联系方式**：
- 私信（知乎私信打开率低，但值得尝试）
- 在其回答下评论引导关注（更自然）
- 如果对方留了公众号/微信，直接加微信

#### 渠道 4：微信群（高效但需要积累）

**目标群类型**：
1. 培训行业交流群
2. HR 从业者群（HR 群里常有培训师潜水）
3. 某个培训领域的专业群（如 "领导力教练群"）

**如何找到这些群**：
- 先加入的培训师朋友圈转发的群
- 培训行业公众号文章底部的社群入口
- 脉脉/知乎上培训师提到的社群
- 搜索 "培训行业交流群" 加入

**入群策略**：
- 不要一进群就发广告，先潜水 3 天
- 主动回答群内问题，提供价值
- 在自我介绍时简短提到 TrainHub
- 私聊感兴趣的培训师

#### 渠道 5：行业论坛/社区

**具体论坛**：
1. 培训杂志社区 / 中国培训网
2. HRoot（人力资源媒体，培训师常活跃）
3. 三茅人力资源网
4. 各类培训认证机构的校友社区（如 ICF 教练社区）

**行动**：关注这些平台上的活跃用户，通过平台私信或公开信息找到微信联系。

#### 早期入驻者激励政策

**不要给钱。** 钱吸引来的是占便宜的人，不是认可你平台价值的人。

**正确的激励**：

1. **"创始培训师" 身份标识**
   - 在个人主页展示 "TrainHub 创始培训师" 徽章
   - 永久享有平台搜索置顶权
   - 这个身份后续越来越值钱

2. **专属服务**
   - 你亲自帮他搭建课程主页（包括文案优化、大纲梳理）
   - 帮他拍摄/设计专业头像和简介页
   - 帮他整理过往培训案例，做成标准化的方案展示

3. **佣金减免**
   - 前 3 笔成交免佣金（后续佣金比例也优惠）
   - 这给了培训师一个"试试看"的理由

4. **社交货币**
   - 在 TrainHub 公众号/社交媒体上给他做专题推荐
   - 帮他做一期 "培训师故事" 的内容

#### 每位培训师入驻的完整流程（Do Things That Don't Scale）

```
1. 微信沟通 → 了解对方背景和需求（30分钟）
2. 收集资料 → 简历、课程大纲、过往案例、照片
3. 你亲手搭建 → 在平台上创建主页和课程（他不需要动手）
4. 发给对方确认 → 让他看效果，提修改意见
5. 上线后通知 → 分享链接，引导他转发到朋友圈
6. 持续跟进 → 每周和他聊一次，问有没有新课程/新案例
```

**时间预算**：每位培训师投入 2-3 小时。10 位 = 20-30 小时。值得。

### 第 8-14 天：从 10 扩展到 30 位

**核心策略：用已有培训师撬动新培训师**

#### 策略 1：培训师转介绍

每位培训师至少认识 10 位同行。这是最高效的获取渠道。

**操作方式**：

```
[培训师姓名]，您入驻后反馈如何？

我想请您帮个忙 -- 您身边有没有 2-3 位同样优秀的培训师朋友？
领域不限，只要是专业靠谱的就行。

我可以同样帮他们免费搭建课程主页。
作为感谢，您推荐的每位培训师成功入驻后，
您在平台的推荐权重会增加（排名更靠前）。
```

**预期转化**：10 位培训师 x 平均推荐 2 位 = 20 位候选人 x 60% 转化 = 12 位新入驻

#### 策略 2：内容驱动

开始在平台和社交媒体上发布入驻培训师的内容：
- "培训师故事" 系列（公众号/知乎/脉脉）
- 入驻培训师的精华课程大纲分享
- 行业观点文章（由培训师口述，你帮写）

**目的**：让还没入驻的培训师看到 "同行都在用"，降低信任门槛。

#### 策略 3：行业活动渗透

参加 1-2 场培训行业线下活动：
- 培训行业年会/论坛
- HR 沙龙
- 培训师认证班结业活动

**关键动作**：
- 不是去摆摊，是去交朋友
- 收集名片 / 加微信 30+ 人
- 活动后 24 小时内逐一跟进

#### 策略 4：培训机构合作

联系 2-3 家小型培训机构，提出合作方案：
- 他们旗下的培训师可以在 TrainHub 上展示
- TrainHub 为他们带来额外的客户线索
- 佣金分成模式

**目标培训机构类型**：
- 5-20 人规模的精品培训公司
- 有自己的培训师团队但缺少线上获客能力
- 不和大平台合作（因为分成太多）

### 第 15-30 天：建立自增长飞轮

到这个阶段，你应该有 30 位培训师，覆盖 TrainHub 的六大品类。

#### 飞轮机制设计

```
更多优质培训师 → 更丰富的课程供给 → 吸引更多企业客户
     ↑                                        ↓
     ← ← ← 更多成交和口碑 ← ← ← ← ← ← ← ←
```

#### 具体动作

1. **建立培训师入驻自助流程**
   - 不再需要你亲手帮每个人搭建
   - 但仍然在 24 小时内人工审核和优化
   - 发送 "入驻指南" 文档给候选人

2. **启动 "培训师推荐官" 计划**
   - 已入驻培训师成功推荐新人 → 获得曝光加权
   - 设立月度 "推荐之星"

3. **内容产出常态化**
   - 每周 2 篇培训师/课程相关内容
   - 知乎回答培训相关问题（引流）
   - 脉脉发布培训行业观察

4. **培训师社群运营**
   - 建立 "TrainHub 培训师" 微信群
   - 每周分享平台数据（浏览量、咨询量）
   - 培训师之间互相交流合作
   - 有新客户需求时在群里发布

## 第二部分：需求侧 -- 企业客户获取

### 第 1-7 天：找到前 5 家种子企业

**核心原则**：不要找大企业，找 100-500 人的中型企业。他们有培训需求但没有成熟的培训供应商体系，决策链条短。

#### 企业筛选标准

**必须��足**：
- [ ] 员工 100-500 人
- [ ] 有明确的 HR 部门（至少 2 人以上）
- [ ] 行业：科技/金融/制造/消费品（这些行业培训预算高）
- [ ] 处于快速成长期（融资后/上市前/业务扩张期）

**加分项**：
- [ ] 最近在招聘 "培训经理" / "组织发展" 岗位（说明正在建设培训体系）
- [ ] CEO/HR 在社交媒体上表达过对人才培养的重视
- [ ] 刚完成融资（有预算）

#### 渠道 1：个人人脉（最优先）

你的朋友、前同事、校友中，谁在管 HR/培训？

**操作**：
1. 梳理微信通讯录，列出所有在 HR 岗位的人
2. 发一条朋友圈（不是广告，是价值提供）：

```
最近在做一件事：帮企业更高效地找到靠谱的培训师。
发现很多 HR 朋友找培训师主要靠朋友推荐，效率低且信息不对称。
如果你正好有培训需求，可以私聊我，帮你免费匹配。
不是广告，真的免费帮忙。
```

3. 私聊 5-10 位 HR 朋友，了解他们当前的培训需求

#### 渠道 2：脉脉（HR/培训负责人）

**搜索关键词**：
1. "培训经理" / "培训总监"
2. "组织发展" / "人才发展"
3. "HRBP"（有培训采购决策权）
4. "企业大学"

**联系话术**：

```
[姓名]您好，

看到您在[公司名]负责培训/人才发展工作。

冒昧打扰 -- 我们做了一个培训师技能市场 TrainHub，
帮 HR 更高效地找到匹配的培训师。
目前有 [X] 位认证培训师入驻，覆盖领导力、销售、数字化转型等领域。

如果您最近有培训需求，我可以免费帮您匹配 2-3 位合适的培训师。
不收取任何费用，纯粹希望验证产品方向。

方便加微信聊聊吗？
```

#### 渠道 3：招聘网站反向查找

在 BOSS直聘 / 猎聘 上搜索正在招聘 "培训经理" 的企业。

**逻辑**：正在招培训经理的企业 = 正在建设培训体系 = 有培训需求但还没有稳定的培训师资源。

**操作**：
1. 搜索 "培训经理" / "培训专员" / "学习发展"
2. 记录招聘企业名称和 HR 联系方式
3. 通过脉脉/LinkedIn 找到该企业的 HR 负责人
4. 用上面的话术联系

#### 渠道 4：HR 社群渗透

加入 HR 交流群（三茅/HRoot/各城市 HR 社群），潜水观察谁在问 "有没有好的培训师推荐"。

**操作**：
- 看到有人问培训需求 → 私聊推荐 TrainHub 上的培训师
- 不要在群里发广告
- 先帮忙解决问题，再引导到平台

#### 渠道 5：企业服务生态合作

找已经在服务中型企业的 SaaS 公司/咨询公司合作：
- 北森/盖雅工场/薪人薪事等 HR SaaS → 他们的客户就是你的目标客户
- 管理咨询公司 → 他们做完咨询后经常被客户问 "有没有配套的培训"
- 企业服务自媒体/社群 → 他们的读者/会员是精准用户

**合作方式**：
- 互相推荐（他们推荐客户给你，你推荐客户给他们）
- 联合内容（共同出一篇 "2026 企业培训趋势" 白皮书）

#### 破冰后的服务流程（Do Things That Don't Scale）

```
1. 了解需求 → 深度电话/视频沟通（30-60分钟）
   - 什么培训主题？
   - 什么人参加？多少人？
   - 预算范围？
   - 什么时间？
   - 之前用过什么培训师？效果如何？

2. 人工匹配 → 从已入驻培训师中挑选 3 位最合适的
   - 不要用算法，用你的判断
   - 给每位推荐理由（为什么这位适合你们）

3. 促成沟通 → 建三方微信群，介绍双方
   - 你在群里当 "媒人"，促成对话
   - 帮培训师准备针对性方案

4. 跟进成交 → 协助价格谈判和合同签订
   - 不急于收佣金，前 5 单免佣金
   - 重点是积累案例

5. 交付跟踪 → 培训结束后收集反馈
   - 企业满意度调研
   - 培训师表现评估
   - 这些数据是未来的竞争壁垒
```

### 第 8-14 天：通过案例裂变

**当你有了第 1 个成功案例，增长就开始了。**

#### 案例包装

每一个成交案例都要包装成以下格式：

```
客户：[公司名]（[行业]，[人数]人）
需求：[一句话描述培训需求]
匹配培训师：[培训师姓名] - [课程名称]
培训效果：[企业反馈的 1-2 句话]
客户评价："[一段真实引用]"
```

#### 案例传播

1. **朋友圈传播**
   - 征得客户同意后，在朋友圈分享案例
   - 让培训师也在朋友圈分享
   - 请企业 HR 在朋友圈推荐

2. **定向推送**
   - 把案例发给同行业的其他企业 HR
   - "您的同行 [公司名] 刚用 TrainHub 完成了 [X] 培训，效果不错，您有类似需求吗？"

3. **内容化**
   - 写成公众号文章
   - 发布在 HR 社区
   - 作为知乎回答的素材

#### 口碑裂变引导

对每一位满意的客户 HR，问这个问题：

```
张总，这次培训效果不错，您有没有 HR 朋友也在找培训师？
我可以帮他们也免费匹配。
```

**预期**：5 家种子企业 x 每家推荐 1-2 家 = 5-10 个新线索

### 每周追踪

| 指标 | 说明 | 目标（第 1 周） | 目标（第 4 周） |
|------|------|-----------------|-----------------|
| 新增培训师 | 本周新入驻 | 10 | 5-8 |
| 新增企业线索 | 本周新咨询 | 5 | 5-8 |
| 撮合数 | 促成培训师-企业沟通 | 0 | 3-5 |
| 成交数 | 签约或完成培训 | 0 | 1-2 |
| 培训师留存率 | 本周活跃培训师 / 总入驻 | 100%（手动维护） | > 70% |

### PMF 判断信号

**已经有 PMF 的信号**（出现 3 个以上就可以加大投入）：
1. 培训师主动把 TrainHub 链接放到自己的简介里
2. 企业 HR 不是你找的，是别人推荐来的
3. 有培训师主动要求入驻（不是你邀请的）
4. 企业 HR 在你没跟进的情况下主动来问新培训师
5. Sean Ellis 测试：问用户 "如果 TrainHub 明天消失，你会？"，> 40% 说 "非常失望"

**还没有 PMF 的信号**（需要调整方向）：
1. 培训师入驻后从不更新/从不登录
2. 企业咨询后不回复
3. 撮合后培训师/企业都不满意
4. 所有用户都是你主动找的，没有自然增长
5. 培训师的课程在平台上和在其他渠道上没有差别

### 转折点判断

| 阶段 | 信号 | 动作 |
|------|------|------|
| 继续手动 | 每周自然增长 < 2 个用户 | 不要放弃，加大手动联系量 |
| 开始半自动 | 每周自然增长 3-5 个用户 | 建立标准化入驻流程，但仍人工审核 |
| 可以规模化 | 每周自然增长 > 10 个用户 | 投入 SEO/内容营销/付费推广 |
| 加速增长 | 月成交 > 10 笔 | 考虑团队扩张，招运营专员 |

## 第四部分：运营陷阱警告

### 陷阱 1：过早追求培训师数量

**错误做法**：第一周就想招 100 个培训师
**正确做法**：10 个高质量培训师 > 100 个注册但不活跃的培训师
**判断标准**：每位培训师的课程主页你看了觉得 "如果我是 HR，我会想联系他" 才算合格

### 陷阱 2：关注虚荣指标

**虚荣指标**：网站 PV、注册量、培训师数量
**真实指标**：撮合数、成交数、复购率
**问自己**：这个数字涨了，收入会跟着涨吗？不会就别看。

### 陷阱 3：过早做功能

**错误做法**：花两周开发在线支付/自动匹配/评价系统
**正确做法**：用微信群+手动匹配搞定前 50 笔交易
**什么时候做功能**：当你发现自己每天花 3 小时做重复性手动操作时

### 陷阱 4：两边同时发力

**错误做法**：同时招培训师和找企业客户
**正确做法**：先搞定供给侧（培训师），再搞需求侧（企业）
**原因**：企业来了发现没有好培训师，就不会再来了。培训师来了暂时没客户，还能等一等。
**节奏**：第 1-3 天 100% 精力在培训师，第 4 天开始 60% 培训师 + 40% 企业

### 陷阱 5：不收费

**错误做法**："先免费积累用户，以后再收费"
**正确做法**：前 5 笔免佣金，第 6 笔开始收 10% 佣金
**原因**：愿意付费的用户才是真用户。免费用户的反馈价值为零。
**Ramen Profitability**：月成交 5 笔 x 平均客单价 30000 x 10% 佣金 = 15000/月。这就是你的拉面钱。

### 陷阱 6：过早外包运营

**错误做法**：找实习生/外包团队来发消息、做客服
**正确做法**：前 3 个月所有用户沟通你亲自做
**原因**：只有你亲自聊，才能感受到真实需���。数据不会告诉你用户的语气和犹豫。
