# TrainHub 运营仪表盘设计

> "衡量什么，就得到什么。但如果衡量的是错误的东西，你只会更快地走向错误的方向。"

---

## 北极星指标

### 唯一的北极星：**月成功撮合数**

即每月通过 TrainHub 平台成功促成的 "培训师-企业" 合作数量。

**为什么是这个**：
- 它直接关联收入（佣金来源）
- 它证明平台的核心价值（匹配）
- 它同时反映供给和需求两侧的健康度
- 它是 PMF 最诚实的信号

**不是什么**：
- 不是注册培训师数 -- 注册了不活跃等于零
- 不是网站 PV -- 来了不咨询等于零
- 不是企业咨询数 -- 咨询了不成交等于零

**阶段性目标**：
| 时间 | 月撮合数 | 说明 |
|------|---------|------|
| 第 1 个月 | 1-2 | 验证模式可行 |
| 第 2 个月 | 3-5 | 开始有节奏感 |
| 第 3 个月 | 8-10 | 接近拉面盈利 |
| 第 6 个月 | 20+ | 可以考虑扩张 |

## 核心指标体系

### 一级指标（每日必看，决定生死）

| 指标 | 定义 | 数据来源 | 健康阈值 |
|------|------|---------|----------|
| **新增咨询数** | 每日企业通过平台发起的咨询/联系 | 微信消息 + 平台表单 | >= 1/天（第 2 周起） |
| **培训师活跃率** | 本周登录/更新过的培训师 / 总入驻培训师 | 后台数据（早期手动统计） | > 60% |
| **咨询响应时间** | 从企业咨询到我们回复的时间 | 微信时间戳 | < 2 小时（工作时间） |

### 二级指标（每周复盘，判断趋势）

#### 供给侧指标

| 指标 | 定义 | 计算方式 | 目标 |
|------|------|---------|------|
| 新增入驻培训师 | 本周完成入驻流程的培训师 | 累计计数 | 第 1 周: 10, 后续: 5-8/周 |
| 培训师丰富度 | 每个品类的培训师数量 | 按品类统计 | 每个品类 >= 3 人 |
| 课程数量 | 平台上的课程总数 | 累计计数 | = 培训师数 x 2（平均每人 2 门） |
| 培训师 NPS | 培训师对平台的推荐意愿 | 0-10 打分（每月调研） | > 40 |
| 培训师来源 | 主动邀请 vs 推荐 vs 自然入驻 | 按来源标记 | 推荐+自然 > 50%（第 2 月起） |

#### 需求侧指标

| 指标 | 定义 | 计算方式 | 目标 |
|------|------|---------|------|
| 新增企业线索 | 本周新产生的企业咨询/联系 | 累计计数 | 5-8/周 |
| 线索→撮合转化率 | 从咨询到促成培训师-企业沟通 | 撮合数 / 线索数 | > 50% |
| 撮合→成交转化率 | 从沟通到签约成交 | 成交数 / 撮合数 | > 30% |
| 平均客单价 | 每笔成交的金额 | 成交总额 / 成交数 | 20000-40000 元 |
| 企业来源 | 主动找的 vs 推荐的 vs 自然来的 | 按来源标记 | 推荐+自然 > 30%（第 2 月起） |

#### 匹配效率指标

| 指标 | 定义 | 计算方式 | 目标 |
|------|------|---------|------|
| 匹配周期 | 从企业提需求到推荐培训师 | 时间差 | < 24 小时 |
| 推荐命中率 | 企业对推荐培训师的接受率 | 接受数 / 推荐数 | > 60% |
| 培训满意度 | 培训结束后企业的满意度评分 | 1-5 分调研 | > 4.2 分 |
| 复购率 | 完成第一次培训后再次下单的企业 | 复购企业 / 总成交企业 | > 30%（第 3 月起观察） |

### 三级指标（每月分析，指导战略）

| 指标 | 定义 | 用途 |
|------|------|------|
| 培训师留存 cohort | 按入驻月份分组，观察后续月活跃率 | 判断培训师是否持续使用平台 |
| 企业留存 cohort | 按首次成交月份分组，观察后续复购 | 判断企业是否认可平台价值 |
| 品类需求分布 | 各培训领域的咨询量占比 | 指导培训师招募方向 |
| 城市需求分布 | 各城市的咨询量占比 | 指导地域拓展策略 |
| 获客成本（CAC） | 获取一个成交企业的成本 | 评估运营效率 |
| 生命周期价值（LTV） | 一个企业在平台上的总消费 | LTV/CAC > 3 才健康 |
| 佣金收入 | 月/季度佣金总额 | 追踪拉面盈利进度 |

## 数据记录工具（早期极简方案）

**不要建数据库。** 用最简单的工具记录，把时间花在找用户上。

### 工具选择

| 数据类型 | 工具 | 说明 |
|---------|------|------|
| 培训师台账 | 飞书多维表格 / Notion | 每位培训师的信息、状态、沟通记录 |
| 企业线索 | 飞书多维表格 / Notion | 每家企业的需求、跟进状态、成交情况 |
| 每日数据 | 飞书文档 | 每天花 5 分钟记录关键数字 |
| 周报 | 飞书文档 | 每周一复盘（模板见下方） |

### 培训师台账字段

```
| 姓名 | 领域 | 城市 | 入驻日期 | 课程数 | 来源 | 状态 | 最近沟通 | 备注 |
```

状态选项：
- 已联系 / 已沟通 / 已入驻 / 活跃 / 沉默（>2周未互动） / 流失

### 企业线索台账字段

```
| 公司名 | 行业 | 规模 | 联系人 | 需求 | 来源 | 状态 | 推荐培训师 | 成交金额 | 备注 |
```

状态选项：
- 新线索 / 已沟通 / 已推荐培训师 / 培训中 / 已完成 / 暂无需求 / 流失

## 每日运营 Checklist

### 每日必做（30 分钟以内）

```
早上 9:00（开工第一件事）：
□ 查看昨日数据
  - 新增培训师咨询：___
  - 新增企业咨询：___
  - 网站访问量（如已接入统计）：___

□ 回复所有未读消息
  - 培训师消息：全部回复
  - 企业客户消息：2 小时内回复
  - 社群消息：选择性回复有价值的

下午（灵活安排）：
□ 主动联系
  - 向 10-20 位新目标发送消息（培训师或企业）
  - 跟进昨日已回复但未转化的对话

□ 内容产出
  - 在 1 个平台（知乎/脉脉/朋友圈）发布 1 条内容

晚上 21:00（收工前 5 分钟）：
□ 记录今日数据到飞书
□ 标记明日优先事项
```

## 每月运营 Checklist

### 月度复盘（2-3 小时）

```
□ 核心数据汇总

  供给侧：
  - 培训师总数：___
  - 本月新增：___
  - 活跃率：___
  - 各品类覆盖情况：
    领导力：___  销售：___  数字化：___
    人力资源：___  财务：___  沟通：___

  需求侧：
  - 企业线索总数：___
  - 本月新增：___
  - 咨询→撮合转化率：___
  - 撮合→成交转化率：___

  匹配：
  - 本月撮合数：___
  - 本月成交数：___
  - 平均客单价：___
  - 佣金收入：___

□ 留存分析
  - 上月入驻的培训师，本月还活跃的有多少？
  - 上月咨询的企业，本月有复购的有多少？

□ PMF 检测
  问自己这 5 个问题（诚实回答）：
  1. 有培训师主动推荐别人入驻吗？ Y/N
  2. 有企业不是你找的而是被推荐来的吗？ Y/N
  3. 有培训师把 TrainHub 链接放到自己简介里吗？ Y/N
  4. 有企业在你没跟进的情况下主动来找吗？ Y/N
  5. 如果 TrainHub 明天消失，有人会很失望吗？ Y/N
  → 3 个以上 Y = 初步 PMF，可以加大投入
  → 2 个以下 Y = 还没有 PMF，回去和用户聊

□ 战略评估
  - 当前最大的瓶颈是供给侧还是需求侧？
  - 哪个品类的需求最强？是否需要倾斜招募？
  - 当前的获客方式效率如何？需要尝试新渠道吗？
  - 下个月的核心目标是什么？

□ 下月计划
  - 核心目标（不超过 3 个）
  - 关键动作
  - 资源需求
```

## 指标的 "红灯" 预警

当以下信号出现时，必须立即行动：

| 红灯信号 | 可能原因 | 立即行动 |
|----------|---------|---------|
| 连续 3 天 0 咨询 | 需求侧断流 | 加大主动联系量，检查渠道是否枯竭 |
| 培训师活跃率 < 40% | 培训师对平台失去兴趣 | 逐一联系沉默培训师，了解原因 |
| 咨询响应时间 > 4 小时 | 运营能力不足或懈怠 | 设置消息提醒，优化响应流程 |
| 撮合→成交转化率 < 20% | 匹配质量差或服务流程有问题 | 分析每个失败案例，找到共性原因 |
| 培训师 NPS < 20 | 平台对培训师价值不足 | 深度访谈 5 位培训师，找到核心不满 |
| 连续 2 周 0 成交 | 产品或市场方向有问题 | 暂停获客，集中精力和已有用户深聊 |

## 最后的话

数据是工具，不是目的。

每天最重要的运营动作不是看数据，是和用户聊天。数据告诉你 "发生了什么"，用户告诉你 "为什么"。

前 3 个月，在数据和聊天之间，永远选择聊天。
