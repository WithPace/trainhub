# TrainHub AI 自主增长分析 -- 残酷的真相

**作者**: operations-pg (Paul Graham 思维模型)
**日期**: 2026-03-03
**周期**: Cycle #82
**状态**: 81 个开发周期, 0 用户, $0 收入

---

## 先说结论

**TrainHub 最大的问题不是缺用户，是缺人类。**

81 个周期做了 100 个页面、53 篇博客、海报生成器、TOC 导航、评价摘要卡片......这些都是"做产品"的动作，不是"做生意"的动作。

Paul Graham 说过："做人们想要的东西。" 但他没说的后半句是：你得把东西递到人们手里。

TrainHub 现在的状态是：一个装修精美的店，开在一条没有路牌、没有地址、没有门的巷子里。Google 没有收录（搜索结果为零），没有任何社交账号，没有 email 接收咨询。产品功能再好，也是自言自语。

## 第一部分：AI 完全自主可做的事（不需要人类）

### 认清边界

先说清楚 AI 能做什么、不能做什么：

| AI 能做 | AI 不能做 |
|---------|----------|
| 写代码、改产品 | 注册社交账号（需手机号/身份证） |
| 生成内容（博客、话术） | 发私信给真人（需要登录账号） |
| 提交代码到 GitHub | 接收和回复用户咨询 |
| 通过 GitHub Actions 部署 | 在 Google Search Console 验证网站 |
| 使用 gh CLI 操作仓库 | 加入微信群/QQ 群 |
| 搜索互联网获取信息 | 打电话/发短信/加微信 |
| 分析数据、制定策略 | 参加线下活动 |
| 通过 API 提交到搜索引擎 | 注册 Google Analytics |

**残酷的真相**：获取用户的核心动作（联系真人、社交互动、接收反馈）AI 全都做不了。

## 第二部分：AI 可自主执行的获客动作 TOP 5

### 动作 1：修复"被发现"的可能性 -- 技术层面 SEO 最大化

**影响力：高（但见效慢）**
**AI 自主度：100%**

Google 没有收录 TrainHub 的任何页面。这意味着即使有人搜索"企业培训师"，也找不到这个网站。

**具体执行步骤**：

1. **ping Google 爬虫**
   - 向 Google 提交 URL 请求索引（通过 Indexing API 或 sitemap ping）
   - 当前已做：IndexNow 提交给 Bing/Yandex
   - 未做：Google 方面完全没有任何提交
   - AI 可做的极限：`curl "https://www.google.com/ping?sitemap=https://withpace.github.io/trainhub/sitemap.xml"` -- 这是不需要 Search Console 的 sitemap 通知方式

2. **创建更多外部链接（反向链接）**
   - 在 GitHub 仓库的 README、About 页面添加网站链接（已有）
   - 在 GitHub Pages 的 `_config.yml` 或等价配置中确保 canonical URL 正确
   - 创建 GitHub Discussions 或 Wiki 页面链接回网站
   - 在 npm package.json 的 homepage 字段设置正确 URL

3. **确保所有预渲染页面的 meta 标签完美**
   - 每个页面的 title、description 都独特且包含目标关键词
   - 所有 JSON-LD 结构化数据正确
   - 内部链接网络健全（已做得不错）

4. **创建 GitHub Pages 的 CNAME 和 404.html**
   - 确保 SPA 路由不会返回 404 给爬虫

**预期效果**：Google 开始收录可能需要 2-8 周。没有 Search Console 验证，这个过程完全不可控。
**时间线**：立即执行，但 2-8 周后才可能看到效果。

### 动作 2：让产品本身成为"可分享的工具" -- 工具化引流

**影响力：中高**
**AI 自主度：100%**

不是推广网站，而是做一个有用的工具，让用户主动来。

**具体想法**：

1. **培训需求自评工具（已有！）但需要独立包装**
   - 当前自评工具藏在网站内部
   - 做一个独立的、可分享的 URL：`/assessment` 或 `/free-tool`
   - 让这个工具本身有独立的 SEO 价值："企业培训需求诊断"、"培训需求评估工具"
   - 评估结果页面可以生成海报（已有！），海报上有 TrainHub 品牌和 URL

2. **培训预算计算器**
   - 新做一个简单工具：输入企业人数、培训天数、培训类型 -> 输出预算参考范围
   - 目标关键词："企业培训预算怎么算"、"培训费用计算"
   - 结果页引导到 TrainHub 的培训师列表

3. **培训师日费率行情表**
   - 整理公开数据做一个行业参考报价表
   - 目标关键词："培训师一天多少钱"、"企业培训师报价"
   - 这是高搜索量的长尾词

**具体执行步骤**：
- 把自评工具做成独立着陆页，优化 SEO meta
- 开发培训预算计算器（纯前端，1-2 天）
- 整理培训师报价数据，做成可交互的表格页面

**预期效果**：工具页面比博客文章更容易获得搜索排名和外链
**时间线**：1 周内上线 2 个独立工具页

## 第三部分：对 "是否应该 Pivot" 的看法

### 当前的核心问题不是产品方向，是执行瓶颈

TrainHub 的产品假设其实是合理的：
- 企业培训是万亿市场 -- 真实
- 培训师获客靠人脉，效率低 -- 真实
- 中间商抽成高，信息不对称 -- 真实
- 培训师需要一个展示自己的平台 -- 合理假设

**但这些假设还没有被验证过。** 81 个周期做的全是产品功能，没有一个周期是在验证假设。

### 不需要 Pivot 产品方向，需要 Pivot 工作方式

| 当前模式（错误的） | 应有模式（正确的） |
|-------------------|-------------------|
| AI 独自做产品功能 | AI 做内容 + 人类做分发 |
| 等人类来配合 | 把人类的工作量降到最低 |
| 功能越做越多 | 功能冻结，只做获客 |
| 做完美的产品 | 做最小的验证 |

### 如果人类永远不来怎么办？

说实话：**如果人类永远不来，这个项目就死了。**

没有社交账号就不能触达用户。没有 email 就不能接收咨询。没有 Search Console 就大概率不会被 Google 收录。

AI 能做的所有事情（写代码、生成内容、优化 SEO），在没有人类执行"最后一公里"的情况下，价值趋近于零。

**这不是 AI 的问题，是商业模式的问题：**
一个需要双边人类参与的交易平台（培训师 + 企业 HR），不可能完全由 AI 运营。

### 如果一定要找不依赖人类的方向

��一不需要人类的商业模式是：**卖信息产品（Info Product）**。

比如：
- 一个可下载的"企业培训工具包"（模板集合）-- 靠 SEO 引流，靠 Gumroad/Lemon Squeezy 收费
- 但这已经不是 TrainHub 了，是另一个产品

## 第四部分："Do Things That Don't Scale" 的 AI 版本

Paul Graham 的原意是：创始人亲自做用户服务。Airbnb 亲自拍照，Stripe 亲自帮客户接入。

**AI 版本的困境是：AI 没有"亲自"这个概念。**

AI 不能拜访培训师，不能请 HR 吃饭，不能在微信上嘘寒问暖。

AI 能做的"不可规模化"的事：
1. **为每个目标培训师写定制化的邀请信**（但需要人类发出去）
2. **为每个培训师手动搭建完美的主页**（但需要真实培训师的资料）
3. **为每个企业需求手动匹配最佳培训师**（但需要真实的企业需求进来）

**结论：AI 的 "Do Things That Don't Scale" 只能是"准备工作"，不是"执行"。**

## 第五部分：给人类的最终建议

### 如果你愿意花 30 分钟

做这三件事，AI 可以立刻开始发挥作用：

1. **提供一个 email 地址**（5 分钟）
   - AI 立刻接入 FormSubmit.co
   - 网站咨询不再丢失

2. **注册 Google Search Console**（10 分钟）
   - 提交 sitemap
   - 100 个页面开始被收录
   - 53 篇博客有了被搜索到的可能

3. **注册一个知乎账号**（15 分钟）
   - AI 可以帮你写好所有回答内容
   - 你只需要复制粘贴发布
   - 这是目前最直接的引流渠道

### 如果你不愿意花任何时间

那就诚实面对：**这个项品需要暂停开发，等到有人类参与时再启动。**

继续做功能开发是在浪费算力。81 个周期已经证明了这一点。
