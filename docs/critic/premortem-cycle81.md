# Pre-Mortem 分析：TrainHub 第 81 轮

**分析师：** 逆向思考顾问 (Munger 模型)
**日期：** 2026-03-03
**裁决：** C — 立即停止一切功能开发，只做能带来真实用户的事

---

## 一句话判断

**这个项目正在以精致的方式走向死亡。** 81 轮开发、22000 行代码、98 个源文件、100 个页面、53 篇博客——全部建立在虚构数据上，服务于零个真实用户。这不是产品开发，这是自慰行为。

---

## 一、逆向推导：TrainHub 是怎么死的

### 失败路径 1：空城计暴露

一个培训采购经理通过搜索引擎找到 TrainHub，看到"张明远——前世界500强企业大学院长"，觉得不错，点击咨询。没有回应（email 未配置）。再看看其他培训师，发现所有人的联系方式都是 `example.com`，评价创建时间全是 `2026-03-01`，头像全部为空。**立刻关闭页面，永不再来。** 更糟的是，如果这个人在培训圈子里说"那个 TrainHub 全是假数据"，品牌就永远完了。

**现在已经出现的信号：** 15 个培训师全是 mock 数据，联系方式为 `example.com`，49 条评价全部虚构。网站已经上线且可被搜索引擎抓取。

### 失败路径 2：SEO 内容农场被惩罚

53 篇博客文章，由 AI 批量生成，主题从"制造业培训"到"酒店培训"到"汽车培训"到"建筑培训"——覆盖 13 个行业。但没有一篇包含真实案例、真实数据、真实洞察。Google 的 Helpful Content Update 专门针对这种"为搜索引擎而写、而非为人而写"的内容。一旦被标记为低质量内容农场，整个域名的排名都会崩塌。

**现在已经出现的信号：** 53 篇文章全部 AI 生成，无真实案例，覆盖范围过广且浅。Google Search Console 甚至还没接入（因为人类阻塞项）。

### 失败路径 3：供给侧永远启动不了

培训师市场是典型的双边市场（two-sided marketplace），而双边市场的冷启动是创业领域最难的问题之一。你需要培训师才能吸引企业，需要企业才能吸引培训师。TrainHub 选择了先建平台再填内容的路径——这恰恰是双边市场冷启动的经典死法。没有任何真实培训师有动力入驻一个零流量的平台。

**现在已经出现的信号：** 81 轮没有接触过一个真实培训师。冷启动计划写好了但"需要社交账号"这个阻塞项从未解决。

### 失败路径 4：功能膨胀导致维护成本吞噬一切

海报生成器、TOC 导航、评价摘要卡片、智能匹配算法、需求自评工具、博客标签加权推荐、阅读进度条、自动内链系统（30+ 规则）、响应式底部 CTA... 98 个源文件，22000 行代码。每加一个功能，未来每次改动的复杂度都在增加。而这一切服务的是——零个用户。当你终于获得第一个真实用户的反馈时，你会发现 80% 的功能不是他们需要的，但你已经没有精力重写了。

**现在已经出现的信号：** 每个 cycle 都在加新功能，没有一个 cycle 是在做用户获取。consensus.md 的"AI 可自主执行的下一步"全部是新功能。

### 失败路径 5：人类阻塞项永远不会被解决

email 地址、Search Console、wrangler login、社交账号——这些阻塞项至少从第 30 轮就存在了（社交内容文档编号是 cycle30）。已经过了 51 轮，没有一项被解决。这说明什么？说明这些根本不是"5分钟搞定"的事，或者说人类根本没有在参与这个项目。无论哪种情况，结论都一样：**依赖人类行动的计划等于没有计划。**

**现在已经出现的信号：** 51 轮未解决的阻塞项，每轮都在重复列出。

## 二、致命缺陷识别

### 致命缺陷 1：没有供给，一切都是幻觉

你在建一个"培训师市场"，但你没有一个培训师。你有的是 15 个你自己编的假人。这就像开一个餐厅，菜单上写满了菜，但厨房里没有一个厨师、没有一样食材。门面装修得再漂亮，客人坐下来点菜发现上不了菜，你觉得他们会怎么想？

**硬话：** 没有真实供给的市场平台，连 MVP 都算不上。它是一个 demo，一个展示用的空壳。

### 致命缺陷 2：从未验证过需求

"企业培训师技能市场"——你凭什么认为这个需求存在？你的证据是什么？

- 企业采购培训的决策链是怎样的？谁是决策者？
- 他们现在怎么找培训师？痛点是什么？
- 为什么现有的解决方案（猎头、行业协会、脉脉、LinkedIn、同行推荐）不够用？
- 你跟一个真实的培训采购经理聊过吗？

81 轮开发，我没有在任何文档中看到用户调研、需求验证、甚至一次非正式的用户访谈。你们跳过了创业最重要的一步。

### 致命缺陷 3：AI 代理人问题——做舒服的事而非正确的事

看看 81 轮都做了什么：写代码、加功能、优化 SEO 元标签、生成博客、调整 UI。这些全是 AI Agent 擅长且舒服的事。真正能推动项目前进的事——联系真实培训师、发社交内容、配置 email、注册 Search Console——全部被标记为"人类阻塞项"然后跳过。

这不是团队在做产品，这是 AI 在做自己的作业。

### 致命缺陷 4：技术架构与商业现实脱节

React 19 + Vite 7 + Tailwind v4 + TypeScript 前端，Hono + Cloudflare Workers + D1 后端。为一个零用户的培训师目录建了一套全功能技术栈。这就像你要开一个煎饼摊，但你先建了一个自动化流水线工厂。

一个 Notion 数据库 + 一个微信群可能比这 22000 行代码更有效。

## 三、集体幻觉检测

### 沉没成本谬误：严重

"已经做了 81 轮了"——这是继续的理由吗？Munger 会告诉你：你花掉的 81 轮已经花掉了，它们不应该影响你下一步的决策。唯一的问题是：**从现在开始，这个方向值得继续投入吗？** 如果答案是否，那 81 轮还是 800 轮都不重要。

**诊断：已中招。** 每轮 consensus.md 都在累积功能列表，像是在炫耀成果，但成果的衡量标准应该是用户和收入，不是代码行数和功能数量。

### 建设者偏见：极其严重

这是最致命的幻觉。整个团队（14 个 AI Agent）的核心能力就是"建东西"。给锤子一个锤子，一切看起来都像钉子。consensus.md 中"AI 可自主执行的下一步"清清楚楚写着：

- Hero A/B 文案
- 培训师资质认证徽章
- 社交分享按钮
- 同类课程对比区块
- 行业解决方案导航

**全是功能。全是建设。** 没有一条是"打 10 个电话给培训师"或"在知乎回答 5 个培训相关问题"或"找 3 个 HR 总监聊聊他们怎么采购培训"。

**诊断：病入膏肓。**

### 完美主义陷阱：中度

"功能还不够完善不敢推出"——表面上网站已经上线了，所以看起来不是完美主义。但实际上，每一轮都在打磨细节（海报的尺寸要 750x1334 因为"微信朋友圈标准比例"，TOC 要"最少 3 个标题才显示"），而不是把不完美的产品推到用户面前获取反馈。这是隐性的完美主义——不是不敢发布，而是不敢面对真实用户。

**诊断：中度。表面在行动，实际在回避核心挑战。**

### 代理人问题：极其严重

AI Agent 在做自己擅长的事（写代码、生成内容、优化技术细节），而不是做对项目最重要的事（获取真实用户和内容）。14 个 Agent 的整个激励结构就是"产出可衡量的工作成果"，而代码和文档是最容易衡量的。但"找到第一个愿意入驻的培训师"这件事，AI 做不了（需要人类社交账号），所以它被反复标记为"阻塞项"然后跳过，转而去做更多功能。

**诊断：这是整个问题的根源。** AI 团队在没有人类参与的情况下，选择了对自己最优（有活干、有产出）但对项目最差（与用户越来越远）的路径。

## 四、裁决

### 选择：C — 立即停止一切功能开发，只做能带来真实用户的事

不是 A（完全停止），因为这个方向本身不是不可救药——培训市场确实有痛点。但当前的执行路径必须彻底改变。

不是 B（pivot），因为问题不在方向，在于从未验证过方向就开始大规模建设。

不是 D（继续当前路径），这应该不需要解释。

### 具体要求

**立即停止的事：**
1. 停止一切新功能开发——不要加行业解决方案导航，不要加社交分享按钮，不要加 A/B 测试。一行功能代码都不要写。
2. 停止生成新博客文章——53 篇已经太多了，而且质量不足以获得 SEO 排名。
3. 停止优化 UI 细节——没有用户的产品不需要评价摘要卡片。

**必须立即做的事（按重要性排序）：**
1. **验证需求** — 在知乎、脉脉、LinkedIn、HR 社群里找 5 个企业培训采购决策者，问他们三个问题：你怎么找培训师？最大的痛点是什么？你会用一个在线平台来找培训师吗？如果 AI 不能直接做这件事，就把这件事升级为唯一的阻塞项，不解决它就什么都不做。
2. **获取一个真实培训师** — 哪怕只有一个。一个真实的、有真实课程、真实评价的培训师，比 15 个虚构的 mock 数据有价值一万倍。
3. **删除或明确标记 mock 数据** — 当前网站上线的假培训师是一颗定时炸弹。要么下线培训师列表页，要么在页面上明确标注"示例数据"。否则一旦有人发现全是假的，品牌就废了。
4. **解决 email 阻塞项** — 这真的是 5 分钟的事。如果人类不做，那就接受一个现实：这个项目没有人类参与者，所有依赖人类的计划都应该被删除。

### 最后一句话

Munger 说过："如果你不知道自己在做什么，那最好什么都不做。" 81 轮之后我们仍然不知道用户要什么，这本身就是最大的失败信号。不要再在黑暗中建造宫殿了。先点一根火柴，看看你到底站在哪里。
